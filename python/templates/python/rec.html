1. Types Of Recursion 

Recursion is mainly of two types:

Direct Recursion
Indirect Recursion
a) Direct Recursion

Direct recursion occurs when a function calls itself directly inside its own body.

This is the most common and simplest form of recursion.

The flow is:

function → same function → same function

The function repeatedly calls itself with a smaller or modified input until the base case is reached.

Most beginner recursion problems use direct recursion.

Examples of use cases:

factorial
sum of natural numbers
Fibonacci
countdown problems

b) Indirect Recursion

Indirect recursion occurs when one function calls another function, and that second function eventually calls the first one again.

The flow is:

function A → function B → function A

So the recursion happens through multiple functions.

This is also called mutual recursion.

It is less common than direct recursion but is useful in advanced logic and algorithm design.

Examples of use cases:

even/odd checking
parser logic
mutually dependent conditions

2. Sub-Types Of Recursion 

These are mostly discussed under direct recursion.
a) Tail Recursion

Tail recursion is when the recursive call is the last statement of the function.

That means after the recursive call, no further work is left.

Flow:

do work → recursive call → end

This is considered memory-efficient in some languages.

But important note for your website:

Python does not optimize tail recursion, so it still uses stack memory like normal recursion.

b) Head Recursion

Head recursion is when the recursive call happens before any processing work.

Flow:

recursive call → do work

The function first keeps calling itself and performs work while returning back.

This is often used in reverse-order outputs.

c) Tree Recursion

This happens when a function calls itself more than one time in a single call.

Flow:

function
   ↙   ↘
 func   func

This creates a tree-like structure of calls.

Commonly used in:

Fibonacci
binary tree traversal
divide and conquer
