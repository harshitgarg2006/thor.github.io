1. Parts Of A Function 
A function mainly consists of:

function name → the name used to call it
parameters → values accepted as input
function body → the logic inside the function
return statement → sends result back

This structure helps in organizing the code into smaller manageable units.
2. Arguments In Functions 
Arguments are the values passed to a function during function call.
They act as the input for the function.

Python supports different types of arguments:

positional arguments
keyword arguments
default arguments
variable length arguments
a) Positional Arguments

In positional arguments, values are assigned according to their position.

The first value goes to the first parameter, the second to the second parameter, and so on.

Here, order is very important.

If the order changes, the result may also change.

This type is used when the sequence of inputs matters.

b) Keyword Arguments

In keyword arguments, values are passed using the parameter names.

Here, order does not matter because Python matches the values by their names.

This makes the code more readable and easier to understand.

It is especially useful when a function has many parameters.

c) Positional-Only Arguments (/)

Python allows some parameters to be positional only.

This is done using / in the parameter list.

All parameters before / must be passed only by position.

Their names cannot be used while calling the function.

This is useful when we want strict positional input.

d) Keyword-Only Arguments (*)

Arguments after * are called keyword-only arguments.

These must be passed using their parameter names.

This improves code clarity and avoids confusion in large functions.

e) Variable Length Arguments (*args)

Sometimes we do not know how many inputs a function will receive.

For this, Python uses *args.

It collects multiple positional arguments into a tuple.

This gives flexibility to accept many values.

Used in:

sum of numbers
dynamic input functions
multiple data processing

f) Dictionary Arguments (**kwargs)

When multiple named arguments need to be passed and the number is not fixed, Python uses **kwargs.

It collects all keyword arguments into a dictionary.

Here:

parameter names become keys
passed values become values

This is useful for flexible named inputs.

Used in:

profile data
settings
forms
configuration functions

3. Packing 
Packing means collecting multiple values into one variable.

Examples:

*args → packs into tuple
**kwargs → packs into dictionary
4. Unpacking 
Unpacking means breaking one collection into separate values.

Python uses:

* for list/tuple unpacking
** for dictionary unpacking

This is commonly used while passing list or dictionary data to functions.
5. Composite Function 
A composite function means one function uses another function inside it.

In simple words, the output of one function becomes the input of another function.

This is called function composition.

Flow:

Function A → result → Function B

This helps in breaking complex tasks into smaller functions.

For example:

one function calculates marks
another function calculates percentage using that result

Composite functions improve modularity and code organization.
